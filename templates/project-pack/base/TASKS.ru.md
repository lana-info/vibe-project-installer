# Задачи

Иди сверху вниз. Каждая задача должна быть маленькой и проверяемой.

## Фаза 1: Определение Продукта

- [ ] T001: Заполнить `PRD.ru.md`
  - Результат: цель, MVP, активные части проекта, feature packs и критерии готовности понятны.
  - Проверка: в PRD нет нерешенных продуктовых вопросов.

- [ ] T002: Проверить PRD в новом AI-контексте
  - Результат: найдены и закрыты пропуски, противоречия и слишком большие куски MVP.
  - Проверка: обновленный PRD не содержит TODO в ключевых требованиях.

## Фаза 2: Первый Рабочий Сценарий

- [ ] T003: Выбрать первый пользовательский сценарий
  - Результат: выбран один маленький сценарий для реализации.
  - Проверка: сценарий можно проверить вручную или тестом.

- [ ] T004: Реализовать первый рабочий кусок
  - Результат: сценарий работает на активных частях проекта: {{ACTIVE_SURFACES}}.
  - Проверка: запущена самая короткая релевантная проверка.

## Фаза 3: Качество

- [ ] T005: Запустить code review
  - Результат: серьезные проблемы исправлены или явно приняты.
  - Проверка: findings имеют file/line и доказательство.

- [ ] T006: Добавить тесты для рискованного поведения
  - Результат: важные auth, data, money, permissions или user flows защищены.
  - Проверка: тесты проходят и покрывают риск.
