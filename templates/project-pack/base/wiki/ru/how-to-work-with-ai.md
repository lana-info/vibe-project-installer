# Как Работать С AI-Агентами Для Кода

Не проси агента построить весь продукт одним запросом. Работай маленькими проверяемыми шагами:

1. Написать или обновить `PRD.ru.md`.
2. Проверить PRD в свежем контексте.
3. Создать или обновить `TASKS.ru.md`.
4. Реализовать одну маленькую задачу.
5. Проверить результат.
6. Запустить code review.
7. Сделать commit.

Обычно хватает 5 основных промптов:

1. Создать или обновить PRD.
2. Проверить PRD.
3. Создать или обновить TASKS.
4. Реализовать одну задачу.
5. Сделать code review.

Шестой промпт, security review, нужен перед продакшеном или если задача затрагивает логин, роли, платежи, файлы или персональные данные.

## Первый Промпт

```text
Прочитай README.md, START_HERE.ru.md, PRD.ru.md, TASKS.ru.md и релевантные файлы.
Пока ничего не меняй.
Скажи, какая следующая самая маленькая безопасная задача, какие файлы затронуты и как ты проверишь результат.
```

## После Утверждения Плана

```text
Реализуй только эту задачу.
Держи изменение минимальным.
Не добавляй несвязанные функции или зависимости.
Запусти самую короткую релевантную проверку.
Сообщи измененные файлы, проверки и что не удалось проверить.
```

## Вариант Через Goal

Используй `goal`, если задача длинная и нужно удерживать один фокус несколько шагов подряд.

Для нового проекта:

```text
Создай goal: подготовить проект к первой маленькой реализуемой задаче.

Прочитай README.md, START_HERE.ru.md, PRD.ru.md, TASKS.ru.md и wiki/ru/how-to-work-with-ai.md.
Если PRD еще сырой, помоги уточнить его через вопросы по одному.
Потом обнови PRD.ru.md, создай или обнови TASKS.ru.md и предложи первую маленькую задачу.
Ничего не устанавливай и не меняй в коде без объяснения зачем.
```

Для работы над задачей:

```text
Продолжай goal.
Возьми следующую открытую задачу из TASKS.ru.md.
Сначала найди связанные файлы и риски.
Потом реализуй минимальное изменение, проверь результат, сделай code review и обнови TASKS.ru.md.
```

Goal завершен, когда результат проверен, `TASKS.ru.md` обновлен, есть понятный следующий шаг и можно делать commit.

## Code Review

```text
Сделай read-only code review текущих изменений.
Ищи реальные баги, регрессии, security/privacy риски, потерю данных и важные отсутствующие тесты.
Верни findings с severity, file/line, evidence, risk, fix и verification.
```
