# Направление Frontend-Дизайна

Цель - не поставить много UI-библиотек, а быстро собрать маленький набор красивых компонентов и аккуратно их доточить.

## Web

Базовый подход: начать с shadcn/ui-style компонентов и потом настроить их под дизайн продукта.

- Используй существующую компонентную систему до добавления новой библиотеки.
- Строй настоящий рабочий экран, а не маркетинговую заглушку.
- Полируй важные компоненты в 2-3 прохода: структура, состояния, spacing, визуальный тон.
- Держи компоненты простыми для правок Codex.
- Используй tokens и variants вместо разовых стилей.

## Mobile

Базовый подход: перед feature-разработкой попросить агента сделать маленькую библиотеку мобильных компонентов под этот проект.

Tamagui не добавлять по умолчанию. Использовать только если есть явная причина и мы готовы к усложнению настройки.

Минимальная мобильная библиотека:

- screen layout;
- buttons;
- inputs;
- cards;
- list rows;
- empty/loading/error states;
- tabs/navigation surfaces;
- modal/sheet patterns;
- typography;
- theme colors and spacing.

## Промпт

```text
Перед реализацией экрана изучи существующие UI components и design tokens.
Не добавляй новую UI-библиотеку.
Предложи минимальный набор компонентов для этого workflow.
Потом реализуй экран в 2-3 прохода качества: структура, визуальная полировка, responsive states.
```
