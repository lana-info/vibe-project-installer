# Project Starter Options

Use this file when deciding what kind of product to vibe-code and which base should be loaded first.

## Current Built-In Bases

- `Vibe mobile + web`: app with web, mobile, backend/API, auth/database patterns, feature packs, deployment planning.
- `Chrome extension`: browser extension based on `JohnBra/vite-web-extension` for React, TypeScript, Tailwind CSS, Vite, and Manifest V3.

## Good Future Bases To Add

- Landing page / marketing site: Astro or Next.js with polished sections, pricing, FAQ, SEO, analytics.
- Internal tool / dashboard: shadcn/ui, tables, filters, forms, charts, auth, admin workflows.
- SaaS app: auth, billing, teams, roles, onboarding, settings, email, audit log.
- AI tool: prompt UI, file upload, job status, history, usage limits, eval prompts.
- Ecommerce/catalog: products, filters, checkout planning, SEO pages, admin content flow.
- Course/membership site: lessons, progress, paywall, library, account area.
- Design system starter: tokens, buttons, inputs, cards, modals, navigation, empty/loading/error states.

## Design Defaults

- Web: start from shadcn/ui-style components and tune the product's own design.
- Mobile: build a small project-specific component library before feature work.
- Extension: keep popup/options/content-script UI focused and permission-light.
